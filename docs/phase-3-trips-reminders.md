# 第 3 阶段：行程与出发提醒

## 目标

让登录用户保存校园行程，并根据真实日期、当前地图路网和最晚到达时间获得建议出发时间与提醒。

## 本阶段交付

- 行程新增、查看、编辑和删除。
- 一次性、每天、每周、每月四种重复规则。
- 当天行程展开与独立路线展示。
- 后端读取最新 `area.json` 路网，计算最短距离和预计步行时间。
- 建议出发时间：`最晚到达时间 - 预计通行时间`。
- 提前提醒分钟数与 `待出发 / 临近出发 / 应立即出发 / 已迟到` 状态。
- 临近出发和应立即出发行程的动态高亮。
- 从行程直接在地图上查看对应路线。

## 日期与重复规则

- 系统业务时区固定为 `Asia/Shanghai`。
- 一次性：仅在开始日期发生。
- 每天：从开始日期起每天发生。
- 每周：在开始日期对应的星期几发生。
- 每月：在开始日期对应的日发生；若当月没有该日，则使用当月最后一天。
- 可选结束日期限制重复行程；未设置时持续有效。

## 路线与时间

- 行程只保存区域、起点地点 ID 和终点地点 ID，不保存一份过期路线。
- 每次读取行程时，后端使用最新路网重新计算最短路径距离。
- 预计步行速度为 `1.2 米/秒`，预计分钟数向上取整。
- 若地点被删除或路径不可达，行程接口会返回明确错误，不能静默给出错误时间。

## 提醒边界

- 当前提醒是网站内提醒：页面每 30 秒刷新当天行程状态。
- 浏览器关闭时不会后台通知用户。
- 系统通知、邮件、短信和推送服务属于部署后的增强能力，不在本阶段实现。

## 验收标准

1. 登录用户可完成行程增删改查，且无法读取或修改其他用户行程。
2. 四种重复规则与结束日期正确展开。
3. 当天行程显示预计通行时间、建议出发时间和提醒状态。
4. 点击行程可在地图查看对应路线。
5. `npm.cmd run check` 与 `npm.cmd run db:upgrade` 通过。
6. 阶段提交已推送 GitHub `master`。
