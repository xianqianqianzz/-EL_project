# 人工标注流程

1. 为区域建立 `data/areas/<area-id>/`。
2. 将该区域唯一底图命名为 `map.png`，后续不得更换尺寸或裁剪。
3. 通过静态服务器打开 `tools/path-editor.html`。
4. 加载 `map.png`；如需继续编辑，再导入已有 `area.json`。
5. 用 `地点 place` 标注大型建筑或重要设施。填写标签后点击图片，工具会自动创建并绑定一个节点。
6. 用 `节点 node` 在道路交叉口、转弯处和必要的连接位置添加节点。节点不填写标签。
7. 用 `连边 edge` 依次点击两个已有节点。工具强制以节点为边的起终点。
8. 导出 `area.json`，与 `map.png` 放在同一区域文件夹。
9. 在 `data/areas/index.json` 登记区域。
10. 运行 `npm run validate:data`，再在网站上实际选择起终点检查路线。

## 人机合作方式

人工负责判断道路是否真实可通行、是否需要转折节点、地点名称是否正确。AI 可以协助检查 JSON 格式、重复 ID、断路、异常长边和文档一致性，但不能代替现场或地图人工判断。
