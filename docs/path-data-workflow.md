# 图片到可行路径数据的人机合作流程

本流程的目标是把地图图片和楼层平面图转成 A* 能使用的路径网络。项目不要求四名组员都会专业 GIS 软件，主流程使用项目自带网页标注工具完成。

## 1. 总体流程

1. D 组整理图片源。
2. A 组维护 `tools/path-editor.html` 标注工具。
3. D 组用标注工具描出可通行路径。
4. B 组运行校验脚本，检查断点、重复 ID、不可达目标。
5. C 组把错误提示整理成人能理解的操作反馈。
6. A/B/C/D 联合抽测示例路线。
7. 审核通过的数据合并到正式 `data/*.json`。

## 2. 图片准备

D 组先把地图图片整理到建议目录：

```text
assets/source-maps/
├── outdoor-xianlin-overview.png
├── yifu-1f.png
├── yifu-2f.png
└── library-1f.png
```

命名规则：

- 室外总图：`outdoor-<campus-or-area>.png`
- 室内楼层：`<buildingId>-<floor>f.png`
- 不确定来源或清晰度不足的图片不要直接进入正式数据，可先放在 `assets/source-maps/pending/`。

每张图需要记录：

- 图片来源
- 对应建筑/楼层
- 是否最新
- 是否有方向问题
- 是否有遮挡、水印、模糊区域

## 3. 标注原则

只标“人能走的线”，不要标装饰性边界。

室外应标：

- 道路中心线
- 广场/开放空间中实际推荐走线
- 校门到路网的连接线
- 建筑入口到最近道路节点的连接线
- 桥、坡道、可通行连廊

室外不应标：

- 建筑外轮廓
- 草坪边界
- 河流、湖岸、装饰线
- 图片文字标注框

室内应标：

- 走廊中心线
- 房间门口到走廊的短连接线
- 楼梯、电梯点
- 入口大厅到主走廊
- 厕所、服务台等设施到最近走廊的连接线

室内不应标：

- 房间中心到房间中心的直线
- 穿墙线
- 穿过桌椅、书架、设备间的线
- 没有门的房间连接线

## 4. 标注工具使用步骤

1. 启动本地服务：

   ```bash
   python -m http.server 8080
   ```

2. 打开：

   ```text
   http://localhost:8080/tools/path-editor.html
   ```

3. 填写 `mapName`、`buildingId`、`floor`。
4. 选择图片文件。
5. 选择节点类型，点击图片添加节点。
6. 选择“画边”模式，依次点击路径折线上的点。
7. 每条边结束后填写 `from` 和 `to` 节点 ID。
8. 对不确定的边设置 `needs-review`。
9. 导出 JSON。
10. 把导出的 JSON 发给 B 组或交给 AI 转换成正式数据。

## 5. AI 可以协助的地方

AI 适合做：

- 根据标注 JSON 生成规范化 `nodes/edges`
- 检查孤立节点、重复 ID、边缺失
- 根据错误信息提出修复建议
- 把标注者写的中文备注整理成 `label/reviewNotes`
- 帮忙生成路线验收清单

AI 不应直接决定：

- 某条路是否真的可走
- 房间门到底在哪
- 楼梯/电梯是否跨层连通
- 模糊图片中看不清的结构

这些必须由人工审核。

## 6. 室外数据审核

每一片室外路网至少检查：

- 校门到附近建筑是否合理
- 建筑到建筑是否沿道路走
- 室外目标是否吸附到最近路网节点
- 路线是否绕过明显不可通行区域

建议验收路线：

- 南门 → 逸夫楼入口
- 南门 → 图书馆入口
- 图书馆 → 仙 I 教学楼
- 地铁站 → 一组团食堂

## 7. 室内数据审核

每栋楼每层至少检查：

- 入口 → 近房间
- 入口 → 远房间
- 入口 → 卫生间
- 同层两个房间之间
- 跨楼层房间之间

室内路径要特别检查：

- 是否从房间门进入走廊
- 是否误穿墙
- 楼梯/电梯是否连到同一个垂直交通系统
- 4F/5F 等特殊不连通区域是否被错误连接

## 8. 合并前检查

每次提交路径数据前运行：

```bash
npm.cmd run validate:data
```

如果脚本报告致命问题，不能合并。  
如果脚本只有 `connections` 单向建议，可以先合并演示版本，但需要在任务清单中记录后续补齐。

## 9. 最小可交付范围

第一阶段不追求全校完整路径，先做：

- 室外：南门、图书馆、逸夫楼、仙 I、仙 II 之间的高质量路径。
- 室内：逸夫楼 1F-3F，入口到 A301、卫生间、计算机实验室。
- 演示路线：南门 → 逸夫楼入口 → 3F A301。

这条路线做扎实，比全校粗糙连线更有说服力。
