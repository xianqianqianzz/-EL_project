# 分阶段路线图

每个阶段必须经过检查、负责人验收和 GitHub `master` 备份后，才能进入下一阶段。

| 阶段 | 范围 | 状态 |
|---|---|---|
| 0 | 稳定现有地图、数据校验、编码规范、基础 CI | 已完成 |
| 1 | 全栈基础、版本化只读区域 API、后端测试 | 已完成 |
| 2 | 用户注册、登录、权限和数据库基础 | 已完成 |
| 3 | 一次性/每日/每周/每月行程、预计通行时间和提醒 | 已完成 |
| 4 | 用户路径修改申请、说明、审核与合并流程 | 已完成 |
| 5 | 管理后台、部署、域名、监控、备份和 GitHub 更新策略 | 已完成 |

阶段范围如需调整，应先修改本文件并经负责人确认。不得因为某项实现方便而提前混入下一阶段业务。
