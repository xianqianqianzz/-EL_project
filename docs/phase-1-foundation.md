# 第 1 阶段：全栈基础

## 目标

在不破坏现有地图和标注流程的前提下，为后续登录、行程、修改申请和后台管理建立稳定入口。

## 本阶段交付

- FastAPI 后端及版本化 `/api/v1`。
- 区域列表、区域 JSON、地图图片和健康检查接口。
- 后端直接托管现有前端。
- 前端优先通过 API 加载区域，API 不可用时回退到静态文件。
- 后端接口自动测试和 GitHub Actions 检查。
- 顶层 Python 依赖固定版本，避免组员与 CI 使用不同版本。
- API、架构、启动和协作说明。

## 本阶段不做

- 不重写地图为 React 或其他框架。
- 不新增登录、数据库、行程或提醒。
- 不开放用户修改主地图。
- 不实现管理后台或 GitHub 自动更新。
- 不修改正式地图数据内容。

## 验收标准

1. `npm.cmd run check` 全部通过。
2. `npm.cmd run dev` 后，`http://localhost:8000/` 可正常规划路线。
3. `http://localhost:8000/docs` 可查看 API。
4. 纯静态服务器启动时地图仍可工作。
5. 阶段提交合并并推送 `master` 后才算建立阶段备份节点。
