# 致组员：项目启动说明

项目已经统一为区域化数据结构。开始工作前，请先阅读：

1. `README.md`
2. `docs/data-format.md`
3. `docs/path-data-workflow.md`
4. `docs/member-guides.md`

当前仙林室外地图的全部正式数据位于：

```text
data/areas/outdoor-xianlin/
├─ map.png
└─ area.json
```

不要再创建或提交 `buildings.json`、`outdoor-nodes.json`、`outdoor-paths.json`、`outdoor-targets.json` 等旧格式文件。

## 每次交付前

1. 确认没有更换或缩放该区域的 `map.png`。
2. 使用标注工具导出规范 `area.json`。
3. 运行 `npm run validate:data`。
4. 在网站中实际选择两个地点，检查路线是否合理。
5. 在提交说明中写明新增或修改了哪些地点、节点和边。
